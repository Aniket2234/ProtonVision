How to Use SafeData Pipeline

1Upload Your Data

Start by uploading your dataset to analyze. SafeData Pipeline supports multiple file formats:

  • CSV files - Comma or semicolon separated values
  • Excel files - Both .xlsx and .xls formats
  • JSON files - Structured data format
  • TSV files - Tab-separated values
  • Parquet files - Columnar storage format
Tip: You can drag and drop files directly onto the upload area, or click to browse and select files.

2Choose Analysis Type

Risk Assessment

Identifies privacy vulnerabilities in your data and measures exposure risk.

Privacy Enhancement

Applies privacy protection techniques like differential privacy or data masking.

Utility Measurement

Measures how much analytical value is preserved after privacy protection.

3Configure Parameters

Each analysis type has specific parameters you can adjust:

  • Attack Simulation Samples: Number of simulated attacks to run (1000-10000)
  • Quasi-Identifier Detection: Automatic or manual selection of identifying columns

  • Privacy Technique: Choose from Differential Privacy, Statistical Disclosure Control, or Synthetic Data
  • Privacy Budget (Îµ): Lower values provide stronger privacy but may reduce data quality

  • Statistical Utility: Measures preservation of statistical properties
  • Machine Learning Utility: Tests how well ML models perform on the data

4Run Analysis & Get Results

Once you've configured the parameters, click "Run Analysis" to start the process. You'll see:

  • Real-time progress - Monitor the analysis as it runs
  • Visual results - Charts and metrics displayed in an easy-to-read format
  • Detailed reports - Comprehensive analysis with recommendations
Download Reports: Generate PDF reports with all results, visualizations, and recommendations for sharing or compliance documentation.

Best Practices

Do
  • Start with Risk Assessment to understand your data's privacy vulnerabilities
  • Clean and prepare your data before uploading
  • Review column names and data types after upload
  • Test different privacy techniques to find the best balance
  • Document your analysis results for compliance
Don't
  • Upload sensitive data without understanding the risks first
  • Use extremely high privacy budgets if strong protection is needed
  • Ignore utility measurement results
  • Skip parameter configuration - defaults may not suit your needs
  • Forget to download and save your analysis reports

Need Help?

If you encounter any issues or have questions:

  • Check the status messages for specific error details
  • Ensure your file format is supported and the file isn't corrupted
  • Try smaller datasets if you're experiencing performance issues
  • Review the analysis parameters to ensure they're appropriate for your data
